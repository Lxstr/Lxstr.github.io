Proposed Project

Overview

A multi-platform software product that will output full sentences based on the selections of a user-defined tick-box form. The user will then be able to save the results and copy/paste or share the output text for use where needed.

Motivation

Health professionals, particularly psychologists and physiotherapists, write reports on the symptoms, appearance and mannerisms of a patient which are follow the same structure each time and result in many hours of wasted writing time that could otherwise be spend with patients. They often have to repeat sentences such as - The patient’s knee was extremely painful or extension and walking. The patient had 70% range of motion. The patient looked tired and had low energy. There are no easily accessible products that can do this. There are over 25 000 members of The APS (The Australian Psychological Society 2020) and 29 000 members of The APA (The Australian Physiotherapy Association 2020).

Description

The app will be a basic three-page application that consists of:

  • A page where a user can set up the tick-boxes and the sentences to go with them. This means typing in the leading text for each series of tick boxes and the names for each tick boxes. It will also allow you to save the template or open others.
  • A page that allows the user to tick the relevant boxes in an easy to use format and clear the tick-boxes when done.
  • A popup or page where the output text is displayed and the user can save, share or export the text quickly to add elsewhere.

Being able to save different templates or output sentences would be a second landmark feature in the development process as it would be highly desirable for a user but not essential, as long as they can easily export results in the meantime.

Tools

The ideal tool for developing this would be the cross-platform programming system Ionic (Ionic 2020), which leverages HTML and Javascript web technologies, making it adaptable and quicker to learn. If it is not possible to learn Ionic in a short enough time, the second preference would be to use Xcode programming and release it only on Mac (Mac Catalyst, Apple 2020), iPad and iOS. Git would be used for version control.

The user will need an android, macOS, iOS or iPadOS running system with up-to-date software.

Skills and Feasibility

Programming skills for development and testing are the main requirement but other skills will allow the app to take form more effectively. Prototyping and user outreach may prevent unnecessary code changes and provide an initial deliverable. Design and presentation skills could help create awareness of the product and advertise it well on a website and the various app stores. The biggest obstacle is likely going to be having to learn app development skills and potentially spending time on this rather that actually producing a product or alternative deliverable. Although this would be very easy to access, given the number free or cheap online courses for Ionic development (Ionic Academy 2020). Therefore, learning this is quite possible in less than 10 hours. Also, there would be no need for external database usage in the Minimum Viable Product, the state and saved templates could be kept on the user's device.

Outcome

A successful app would allow many professionals to save work hours at a small cost and would help improve the world. For example, psychologists would be able to quickly record the manner of the patient and focus on the main purpose of the appointment. However, even a prototype, customer engagement and specification would allow development to continue at a later time if it is found to be viable.
