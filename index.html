My Journey Into IT

In 2016 I decided to put together an iOS application to automate fuel planning for the AP3C, which was the aircraft I was flying as a pilot. It was a legacy aircraft where the fuel planning was still done with various lookup charts and tables, sometimes different versions of the same thing. I found I could save many minutes per day and even reduce the likelihood of error by translating the tables into equations with python and then creating a swift algorithm for use by the app. I enjoyed both the programming side and creating a user interface that made the process as rapid as possible and error safe.

This helped me I realise I would enjoy being part of teams that can help automate work processes and make life easier for people. I am also fascinated by the area of cryptography and the possible applications for security and data integrity. My dream would be to help build a product with an invested and passionate user base where I could be a conduit between operators, engineers and developers.

Proposed Group Project

The project will be a iOS or ionic (cross platform) app that allows the user to automatically convert a series of tick boxes (that they can define) into sentences.

I have heard from a psychologist who regularly needs to write reports on the appearance and mannerisms of a patient which are very repetitive and follow the same structure each time. I suspect that there are other professions that have similar report writing needs and would be willing to pay for the time savings this software could give.

The features required are:
  • A templating page for setting up the tickboxes. This includes typing in the leading text for each series of tick boxes and the names for each tick boxes. It should also allow you to save the template.
  • A page that allows the user to tick the relevant boxes in a easy to use format and clear the selections when done
  • A popup or page where the output text is displayed and the user can share or export the text quickly to add elsewhere
  • An open and save button with associated save data storage
In one option, we would use a cross platform programming tool such as ionic or flutter that can run on all platforms. The second preference would be to use iOS programming and run it only on Mac, iPad and iOS. The amount of previous experience with this software would be a major factor in the groups choice. In both instances, we would collaborate on GitHub.

Programming skills for development and testing are the main requirement but other skills will allow the app to take form more effectively. Prototyping and user outreach may prevent unnecessary code changes and provide an initial deliverable. Design and presentation skills could help create awareness of the product and advertise it well on a website and the various app stores. In addition, coordination skills will greatly increase the efficiency of work. These skills are quite feasible within our group and a reasonable deliverable would be a detailed specification and prototype in Invision Studio or Sketch. However, an application may be possible.

The biggest obstacle is likely going to be having to learn app development skills and potentially having multiple people spending there time on this rather that actually producing a product or deliverable.

A successful app would allow many professionals to save work hours at a small cost and would help improve the world.
